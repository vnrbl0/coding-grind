CSS Specificity & Cascade

CSS specificity is a set of rules that determines which CSS rule applies when multiple rules could apply to the same element. It is calculated based on the types of selectors used in the rule.

Here's a quick overview of how specificity is calculated:

  • Inline styles (e.g., style="color: red;") have the highest specificity.
  • ID selectors (e.g., #myId) have a higher specificity than class selectors.
  • Class selectors (e.g., .myClass), attribute selectors (e.g., [type="text"]), and pseudo-classes (e.g., :hover) have a lower specificity than ID selectors.
  • Type selectors (e.g., div, p) and pseudo-elements (e.g., ::before) have the lowest specificity.
  • Universal selector (*) has no specificity.
  • Inserting important in a rule (e.g., color: red !important) will override any other rules, regardless of specificity.

When multiple rules apply to the same element, the rule with the highest specificity will be applied. If two rules have the same specificity, the one that appears last in the CSS will take precedence.

Understanding specificity is crucial for writing effective CSS and avoiding conflicts between styles.

Cascade Calculation

The CSS cascade is the process by which styles are applied to elements based on their specificity and source order. The cascade takes into account the following factors:

  • Importance: Styles marked with !important will override all other styles.
  • Specificity: More specific selectors will override less specific ones.
  • Source order: If two rules have the same specificity, the one that appears later in the CSS will be applied.
  • Origin: Styles can come from different sources, such as user agent styles, user styles, author styles, and inline styles. Author styles have the highest priority.

Understanding the cascade is essential for managing styles effectively and ensuring that your CSS behaves as expected.

Cascade Values in numbers

Specificity is calculated using a four-part value:

  • Inline styles: 1,0,0,0
  • ID selectors: 0,1,0,0
  • Class selectors, attributes, and pseudo-classes: 0,0,1,0
  • Type selectors and pseudo-elements: 0,0,0,1
  • Universal selector: 0,0,0,0

For example, if you have a rule with an ID selector and a class selector, the specificity would be calculated as follows:

ID selector: 0,1,0,0 + Class selector: 0,0,1,0 = 0,1,1,0

The specify value of each selectors

  • Universal selectors : 0
  • Type selectors : 1
  • Class selectors : 10
  • ID selectors : 100
  • Inline styles : 1000

    • Understanding these values helps you predict which styles will be applied to an element based on the specificity of the selectors used.

    Practical Example

    Consider the following CSS rules:

            .example {
            color: blue; /* Specificity: 0,0,1,0 */
        }
        #example {
            color: red; /* Specificity: 0,1,0,0 */
        }
        div {
            color: green; /* Specificity: 0,0,0,1 */
        }
        Inline style /* Specificity: 1,0,0,0 */

    In this case, the inline style will take precedence over the ID selector, which will take precedence over the class selector, and finally the type selector. The final color of the element will be orange.
